# X Bookmark 週次収集スキル（v2）

毎週火曜0:00に自動実行。新着ブックマークを収集し、以下の2ファイルを作成する。

- `bookmarks_MMDD-MMDD.md` — 全文統合ファイル（収集順に羅載）
- `bookmarks_index_MMDD-MMDD.md` — カテゴリ別インデックス

## ステップ1: ブックマーク収集

```bash
cd ~/test2 && python3 scripts/fetch_bookmarks_v2.py
```

実行後、出力の最終行 `OUTPUT_FILE:/path/to/file.md` からファイルパスを取得する。
「新着ブックマークはありません」と出た場合はここで終了。

## ステップ2: インデックスファイルを作成

ステップ1で作成された統合mdファイルを読み込み、以下の形式でインデックスファイルを作成する。

**インデックスのファイル名:** 統合ファイルと同じ日付範囲で `bookmarks_index_MMDD-MMDD.md`

**インデックスの構成:**

```markdown
# Xブックマーク インデックス YYYY/MM/DD 〜 MM/DD

**合計：N件**

---

## カテゴリー別 目次

### A. [カテゴリ名]（N件）

| # | タイトル概要 | 投稿者 |
|---|---|---|
| A-1 | **概要** — 詳細説明 | @username |
...

---

## 全体サマリー

| カテゴリ | 件数 |
|---|---|
| A. カテゴリ名 | N件 |
| **合計** | **N件** |

---

## よく出るキーワード

- **キーワード** — 関連する投稿の傾向説明
...
```

**カテゴリ分類の方針:**
- 内容からテーマを読み取り、自然なカテゴリに分類する
- カテゴリ化が難しい場合は「その他」にまとめる
- カテゴリは5〜8個程度が目安（件数が少なければそれ以下でもよい）

インデックスファイルは統合ファイルと同じディレクトリ（`~/test2/bookmarks/`）に保存する。

## ステップ3: Google Driveにアップロード

統合ファイルのみアップロードする（インデックスファイルはアップロード不要）。

```bash
rclone copy [統合ファイルのパス] gdrive:X-Bookmarks-Weekly/ --progress
```

アップロード成功を確認したら完了を報告する。

## ステップ4: Discord通知

`~/test2/.discord_webhook` が存在する場合のみ実行する。

```bash
WEBHOOK_URL=$(cat ~/test2/.discord_webhook 2>/dev/null)
```

ファイルが存在しない場合はスキップして完了を報告する。

存在する場合は、ステップ1で取得した件数とファイル名を使って通知を送る：

```bash
curl -s -X POST "$WEBHOOK_URL" \
  -H "Content-Type: application/json" \
  -d "{\"content\": \"週次ブックマーク収集完了\\n件数: ${COUNT}件\\nファイル: ${FILENAME}\"}"
```

`COUNT` はステップ1の出力から取得した件数、`FILENAME` は統合ファイル名（例: `bookmarks_0407-0413.md`）を使う。

送信成功（HTTPステータス2xx）を確認したら完了を報告する。

## ステップ5: INBOX に日付フォルダーを作成

統合ファイル名（例: `bookmarks_0414-0420.md`）から週の開始日を取得し、フォルダーを作成する。

```bash
# ファイル名から開始日を取得（例: 0414 → 2026-04-14）
BASENAME=$(basename [統合ファイルパス] .md)   # bookmarks_0414-0420
START_MMDD=$(echo $BASENAME | sed 's/bookmarks_\([0-9]*\)-.*/\1/')  # 0414
YEAR=$(date +%Y)
FOLDER_DATE="${YEAR}-${START_MMDD:0:2}-${START_MMDD:2:2}"  # 2026-04-14
INBOX_DIR=~/dotfiles/INBOX/bookmarks_analysis/${FOLDER_DATE}
mkdir -p "$INBOX_DIR"
```

## ステップ6: 全ブックマーク精読 → bookmarks_analysis.md 生成

### 精読の手順

1. ステップ2で作成したインデックスファイルを読み込み、全カテゴリを把握する
2. 統合ファイル（全文）を読み込み、**インデックスの全カテゴリを1件も飛ばさず**精読する
3. 各トピックについて以下の形式で分析を書く

### 分析ルール（必須）

**ルール1：全トピック均等精読**
- インデックスに載っているカテゴリは件数が1件でも全て分析する
- 「件数が少ない＝重要度が低い」と判断してスキップしない

**ルール2：重要度の判定基準は多軸で評価する**

| 軸 | 判定基準 |
|---|---|
| 既存環境との接続性 | 今使っているツール・スキルと繋がるか |
| 即効性 | 設定・インストールなしで今日から使えるか |
| 波及範囲 | 一度導入すると複数の作業に効くか |
| 公式情報の有無 | Anthropic等の一次情報が存在するか |

**ルール3：分析漏れチェック**
- 全トピックの分析を書き終えたら、インデックスの全カテゴリと照合する
- 未分析のカテゴリがあれば必ず追加してから推奨アクションを書く

### bookmarks_analysis.md の形式

```markdown
# ブックマーク分析 YYYY-MM-DD

- 対象期間：YYYY-MM-DD 〜 YYYY-MM-DD
- 総件数：N件

---

## [トピック名]（N件）

**概要：** 何をするものか1〜2文で説明

**自分への影響度：** 高/中/低 — 理由（今の環境・使い方に関係あるか）

**すぐ使えるか：** 今日 / 今週 / 今月 / 様子見 — 理由

**既存との重複：** あり/なし/部分あり — 理由（すでにスキル・設定・ツールとして持っているか）

---

（以降、全カテゴリ同形式で繰り返す）

---

## まとめ：推奨アクション

| 優先度 | やること | 理由 |
|---|---|---|
| **今すぐ** | 具体的なアクション | 推奨する理由 |
| **今すぐ** | 具体的なアクション | 推奨する理由 |
| **今週中** | 具体的なアクション | 推奨する理由 |
| **今月中** | 具体的なアクション | 推奨する理由 |
| **様子見** | 具体的なアクション | 推奨する理由 |
```

生成したファイルは `$INBOX_DIR/bookmarks_analysis.md` に保存する。

## ステップ7: 重要トピックの深掘りMD生成

以下の条件を**1つ以上**満たすトピックについて、深掘りMDを生成する。

| 条件 | 基準 |
|---|---|
| 同一テーマのブックマーク数 | 3件以上 |
| 公式リリース・新機能発表 | Anthropic等の一次情報あり |
| 既存環境への接続性 | 今のスキル・設定と直接つながる |

深掘りMDの内容：
1. WebSearchで公式情報（Anthropic公式等）を調査
2. ブックマーク内容と公式情報を統合してまとめる
3. 機能詳細・利用条件・実際に使った声・推奨アクションを含める

ファイル名は `{トピック名_スネークケース}.md`（例: `claude_design.md`）とし、`$INBOX_DIR/` に保存する。

## ステップ8: git commit

```bash
cd ~/dotfiles
git add INBOX/bookmarks_analysis/${FOLDER_DATE}/
git commit -m "週次ブックマーク分析：${FOLDER_DATE}"
```

コミット成功を確認したら全ステップ完了を報告する。

## エラー対応

- **セッション切れ**: `cd ~/test2 && python3 scripts/save_session.py` を実行してログインし直す
- **rclone失敗**: `rclone listremotes` でリモート名を確認する
- **新着0件**: 何もせず終了（正常動作）
- **git commit失敗**: `cd ~/dotfiles && git status` で状態を確認する
